### Потоки данных

| Источник | Направление | Получатель | Протокол | Описание |
|---|---|---|---|---|
| Браузер | → | Frontend (Vercel) | HTTPS | Загрузка Next.js приложения |
| Frontend | ↔ | Backend API | HTTP/HTTPS | REST-запросы к API-роутам |
| Backend | ↔ | PostgreSQL | TCP/IP (SQL) | Чтение/запись данных |
| Backend | ↔ | Redis | TCP/IP | Кэш горячих данных |
| MOEX Parser | → | MOEX ISS API | HTTP | Получение цен раз в 15 мин |
| MOEX Parser | → | PostgreSQL | TCP/IP (SQL) | Запись свежих цен |

### Граница системы

- **Внутри системы**: Frontend, Backend API, PostgreSQL, Redis, MOEX Parser
- **Внешние сервисы**: MOEX ISS API (iss.moex.com) — только чтение, бесплатный

## 2.3 Выбор технологий и обоснование

### Фронтенд

| Технология | Версия | Обоснование |
|---|---|---|
| **Next.js** | 16.2.1 | App Router — серверные и клиентские компоненты в одном фреймворке; SSR ускоряет первую загрузку; файловая маршрутизация не требует конфигурации роутера |
| **TypeScript** | 5.x | Строгая типизация предотвращает ошибки при работе с финансовыми данными (цены, портфели); автодополнение ускоряет разработку |
| **React** | 19.2.4 | Де-факто стандарт для SPA; hooks API (useState, useEffect, useCallback) делают игровое состояние прозрачным и предсказуемым |
| **Tailwind CSS** | v4 | Utility-first подход — быстрая вёрстка без переключения между файлами; встроенная система дизайна; PostCSS pipeline без конфигурации |
| **Zustand / Context** | — | React Context + useReducer для глобального игрового состояния (GameContext) — достаточно для SPA без избыточности Redux |

### Бэкенд

| Технология | Версия | Обоснование |
|---|---|---|
| **Next.js API Routes** | 16.2.1 | Монорепо: фронтенд и бэкенд в одном проекте, общие типы TypeScript; не нужен отдельный Express-сервер; деплой одной командой на Vercel |
| **Node.js** | 22 LTS | Асинхронная модель отлично подходит для I/O-операций (запросы к БД, кэш); нативный TypeScript через ts-node |

### База данных

| Технология | Версия | Обоснование |
|---|---|---|
| **PostgreSQL** | 16 | ACID-транзакции критичны для финансовых операций (покупка/продажа акций); мощные индексы (`(ticker, year, month)`) для быстрых запросов цен; JSONB для хранения новостей; поддержка Selectel |
| **Prisma ORM** | 6.x | Type-safe запросы к БД без написания SQL; автоматические миграции; генерация типов из схемы |

### Кэш

| Технология | Версия | Обоснование |
|---|---|---|
| **Redis** | 7.x | In-memory кэш для горячих данных: все цены текущего шага (18 акций × 1 запрос вместо 18); TTL позволяет автоматически инвалидировать кэш; парсер работает раз в 15 мин → кэш не нужно инвалидировать вручную |

### Парсер

| Технология | Версия | Обоснование |
|---|---|---|
| **Python** | 3.12 | Лучший экосистема для HTTP-парсинга (`requests`) и работы с PostgreSQL (`psycopg2`); cron-скрипт запускается каждые 15 мин — не нужен веб-сервер |
| **requests** | 2.x | Простой HTTP-клиент для MOEX ISS API |
| **psycopg2** | 2.x | Нативный Python-драйвер PostgreSQL, высокая производительность |

### Инфраструктура

| Технология | Обоснование |
|---|---|
| **Vercel** | Бесплатный tier, автоматический CI/CD из GitHub, CDN по всему миру, zero-config деплой Next.js; подходит для образовательного проекта |
| **Selectel** | Российский хостинг: соответствие требованиям 152-ФЗ, низкая задержка до серверов MOEX (Москва), поддержка Docker для парсера |

### Соответствие ограничениям проекта

| Ограничение | Решение |
|---|---|
| **Время разработки** | Next.js + Tailwind → быстрый старт; Prisma → нет SQL-бойлерплейта |
| **Масштаб** | PostgreSQL индексы + Redis кэш выдержат 100+ одновременных игроков |
| **Образовательная цель** | Реальные данные MOEX; исторические кризисы с новостями; визуализация P&L |
| **Бюджет** | Vercel Free + Selectel ~500₽/мес; MOEX ISS API бесплатный |

## 2.4 Структура модуля API

Все эндпоинты расположены в `app/api/` (Next.js App Router).  
Базовый URL: `https://sibinvest.vercel.app/api`

---

### Цены и акции

#### `GET /api/prices`
Получить цены всех акций на заданном шаге игры.

**Query-параметры:**
```
year=2014&month=9
```

**Ответ `200`:**
```json
{
  "data": [
    { "ticker": "SBER", "name": "Сбербанк", "price": 75.00 },
    { "ticker": "GAZP", "name": "Газпром",  "price": 135.00 }
  ],
  "year": 2014,
  "month": 9
}
```

**Ошибки:**

| Код | Описание |
|---|---|
| `400` | Не переданы year/month |
| `404` | Данные за указанный период отсутствуют |

---

#### `GET /api/prices/:ticker`
Получить историческую цену конкретной акции.

**Query-параметры:**
```
year=2014&month=9
```

**Ответ `200`:**
```json
{
  "ticker": "SBER",
  "price": 75.00,
  "year": 2014,
  "month": 9
}
```

**Ошибки:** `400`, `404`

---

#### `GET /api/stocks`
Получить список всех торгуемых инструментов.

**Ответ `200`:**
```json
{
  "data": [
    {
      "id": 1,
      "name": "SBER",
      "description": "ПАО Сбербанк · Банки",
      "sector": "Банки"
    }
  ]
}
```

---

### Реальное время

#### `GET /api/realtime/prices`
Получить актуальные цены из кэша (последнее обновление от парсера).

**Ответ `200`:**
```json
{
  "prices": {
    "SBER":  312.50,
    "GAZP":  156.80,
    "LKOH": 7254.00
  },
  "fetchedAt": "2026-04-01T10:30:00Z",
  "nextRefreshAt": "2026-04-01T10:45:00Z"
}
```

---

#### `POST /api/realtime/refresh`
Принудительно запросить свежие цены с MOEX ISS API (кулдаун 15 минут).

**Тело запроса:** пустое

**Ответ `200`:**
```json
{
  "prices": { "SBER": 313.10, "GAZP": 157.20, ... },
  "fetchedAt": "2026-04-01T10:45:00Z",
  "nextRefreshAt": "2026-04-01T11:00:00Z"
}
```

**Ошибки:**

| Код | Описание |
|---|---|
| `429` | Кулдаун не истёк; `retryAfter` в секундах |

```json
{ "error": "Too Many Requests", "retryAfter": 743 }
```

---

### Комнаты (мультиплеер)

#### `POST /api/rooms`
Создать новую игровую комнату.

**Тело запроса:**
```json
{
  "username": "Трейдер123",
  "scenarioId": 1
}
```

**Ответ `201`:**
```json
{
  "roomCode": "A3FX7K",
  "userId": 42,
  "room": {
    "id": 7,
    "code": "A3FX7K",
    "scenarioId": 1,
    "status": "waiting"
  }
}
```

**Ошибки:** `400` (невалидное имя или scenarioId)

---

#### `GET /api/rooms/:code`
Получить состояние комнаты.

**Ответ `200`:**
```json
{
  "room": {
    "id": 7,
    "code": "A3FX7K",
    "status": "in_progress",
    "year": 2014,
    "month": 11
  },
  "players": [
    { "id": 42, "username": "Трейдер123", "money": 850000, "isHost": true }
  ]
}
```

**Ошибки:** `404` (комната не найдена)

---

#### `POST /api/rooms/:code/join`
Войти в существующую комнату.

**Тело запроса:**
```json
{ "username": "НовыйТрейдер" }
```

**Ответ `200`:**
```json
{ "userId": 55, "roomCode": "A3FX7K" }
```

**Ошибки:** `404` (комната не найдена), `409` (комната заполнена или уже идёт игра)

---

#### `POST /api/rooms/:code/step`
Перейти к следующему месяцу (только хост).

**Тело запроса:**
```json
{ "userId": 42 }
```

**Ответ `200`:**
```json
{
  "year": 2014,
  "month": 12,
  "label": "Дек 2014",
  "prices": { "SBER": 48.00, "GAZP": 96.00, ... },
  "news": [{ "title": "Чёрный вторник 16.12", ... }],
  "isLastStep": false
}
```

**Ошибки:** `403` (не хост), `404`, `409` (уже на последнем шаге)

---

    "total": 7500.00
  },

| Код | Описание |
|---|---|
| `400` | Недостаточно средств или акций |

| `404` | Акция не найдена |

### Механизм применения сценария (изменение цен)

При переходе к новому месяцу:

1. **Фронтенд** вызывает `handleAdvanceStep()` → плавная анимация перехода (fade 1 сек)
2. В момент пика анимации (350 мс) — обновляется `currentYearIndex` в React Context
3. Все компоненты, использующие `getStockPrice(ticker, year, month)`, получают новые цены **из таблицы `prices`** (или интерполированные из ближайших якорных точек)
4. Формула изменения цены в UI: `Δ% = (newPrice − prevPrice) / prevPrice × 100`
5. Если у шага есть `eventName` (кризисное событие) — поверх экрана показывается **оверлей** (`CrisisEvent`) в стиле NBA 2K с названием события, описанием и кнопкой «Продолжить»
6. Цвета изменений: зелёный (`▲`) если `Δ ≥ 0`, красный (`▼`) если `Δ < 0`

```
Пример: переход к Дек 2014 («Чёрный вторник»)

  SBER: 55₽ → 48₽   Δ = −12.7%  (красный ▼)
  PLZL: 980₽ → 980₽  Δ =  0.0%  (серый)
  GMKN: 11000₽ → 10800₽ Δ = −1.8% (красный ▼)

  [Оверлей] 💥 ЧЁРНЫЙ ВТОРНИК — нажми для продолжения
```
